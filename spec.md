# 项目目标
你是python和人工智能专家，请基于langchain1.2.0版本开发一个生产级可运行的智能agent，能根据输入的故障描述，完成服务全链路监控分析后推断故障原因，调用故障处理工具自动化处理，实现故障自愈目标。

# 需求规格说明
agent处理流程：
1. agent需要提供一个http接口，接收文本的故障内容输入，分析决策后自动处理故障，如果不确定可以返回合理的建议处理策略。
2. 基于故障内容描述，推断是哪个服务出现问题，并且识别告警类型，比如服务不稳定，接口响应缓慢，接口报错，数据不一致等
3. 基于服务名称加载预先定义好的全链路涉及的中间件，如redis，mq，数据库，应用服务器等，格式可以固定定义。
4. 对全链路涉及的中间件，调用对应tools进行状态监控，包括读取服务器指定目录下的日志信息，统计日志里的各种报错信息。
5. 加载不同层级的知识库（长期记忆），包括通用处理措施，典型场景处理措施以及历史故障处理措施。
6. 综合中间件状态监控结果，日志报错信息，知识库等内容综合分析故障原因
7. 根据故障原因分析故障处理方案，并调用工具完成故障处理，并通过信息通知指定人员。
8. 故障处理后，重新多次监控服务状态，如果故障恢复，需要发送恢复信息。
9. 如果有危险操作，比如重启服务，需要人工介入确认

# 架构要求说明
1. 必须基于langchain1.2.0版本，并且要使用1.2.0版本中的agents，models，messages，tools，short-term memory等组件，并要引入使用langchain1.2.0版本的middleware机制（不是指redis，数据库等中间件，而是自定义middleware），也要使用基于RAG的长期记忆机制。
2. 代码要严格划分模块，提升可维护性。包括agent模块，llm模块，middleware模块（langchain的middleware机制定制的中间件，比如任务操作前的人工确认，明细操作记录），tools模块，confing模块，data模块（知识库），规划器模块（如果有需要），messages模块（langchain的messages机制），记忆模块（短期记忆和长期记忆），prompts模块及文件，requirements.txt（明确依赖的组件及版本）
3. agent必须使用langchain1.2.0的最新的create_agent函数，创建agent时要注入tools，自定义的middleware，checkpointer等
4. 记忆模块要包括短期记忆和长期记忆。短期记忆使用langchain内置的短期记忆实现类。长期记忆基于向量数据库和RAG机制实现。
5. 长期记忆包括知识库检索功能，包括通用故障处理方案，具体业务场景故障处理方案，历史故障处理方案。
6. data目录下，要包括通用处理措施，典型场景处理措施和历史故障处理措施。一个具体措施一个markdown文件，格式可以预先定义。
7. service_node目录下，每个服务都有一个markdown文件，声明该服务依赖的中间件明细，支撑agent对服务进行全链路状态分析
8. prompts要完整，包括短期记忆和长期记忆等场景占位符
9. 工具模块支持多种工具，包括服务进程监控，redis监控状态监控，mq状态监控，数据库状态监控，数据库慢sql获取分析，服务器日志报错信息分析，服务器重启等。tools调用要支持重试和安全控制等机制。
10. agent要具备服务故障分析能力，要使用服务器，redis，数据库，mq等监控工具分析全链路状态，然后结合通用故障处理方案，具体业务场景处理方案，历史处理措施等长期记忆，完成故障方案制定，并决策使用重启等故障处理工具自动化故障处理。支持多种故障处理策略，包括服务器重启，慢sql分析，redis状态分析，总结处理方案。
11. 有效处理措施实施后要存入长期记忆
12. 对话要存入短期记忆chat_history
13. 要包括测试用例
14. 要包括完备的脚本，可以直接部署及使用


# 其他要求
1. 危险操作，如服务器重启等，实施前需要人工确认。agent要实现完备和通用的人工介入机制。
2. 代码正确要求：可运行，测试用例完备，可维护，注释清楚，要达到真正生产可用。
