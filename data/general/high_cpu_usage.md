# 服务 CPU 使用率过高处理方案

## 适用场景
服务器或应用进程 CPU 使用率持续超过 80% 的通用处理方案。

## 故障现象
- 服务器 CPU 使用率居高不下（>80%）
- 系统响应变慢
- 服务请求处理延迟增加
- load average 异常升高

## 根因分析方向
1. **代码层面**：死循环、CPU 密集型计算、正则表达式回溯
2. **GC 频繁**：JVM 应用堆内存不足导致频繁 Full GC
3. **线程池打满**：大量线程上下文切换
4. **外部依赖慢**：数据库慢查询、Redis 超时导致请求堆积
5. **流量突增**：业务流量超预期增长

## 处理步骤

### 1. 快速定位高 CPU 进程
```bash
# 查看 CPU 使用率最高的进程
top -bn1 | head -20

# 定位高 CPU 线程（Java 应用）
top -Hp <PID>

# 将线程 ID 转为十六进制后查找堆栈
printf '%x' <TID>
jstack <PID> | grep -A 20 "<hex_tid>"
```

### 2. 分析 GC 情况（Java 应用）
```bash
# 查看 GC 统计
jstat -gcutil <PID> 1000 10

# 如果 Full GC 频繁，检查堆内存
jmap -heap <PID>
```

### 3. 限制 CPU 使用
```bash
# 临时降低进程优先级
renice +10 <PID>

# 使用 cpulimit 限制 CPU（需安装）
cpulimit -p <PID> -l 50
```

### 4. 根本性修复
- 优化 CPU 密集型代码逻辑
- 增加 JVM 堆内存（-Xmx）减少 GC 频率
- 扩容服务器或增加服务实例
- 启用限流熔断，防止流量突增

## 预防措施
- 设置 CPU 使用率告警阈值（70%、85%）
- 进行压力测试，确定服务容量上限
- 配置自动扩缩容策略
